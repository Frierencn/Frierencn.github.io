在重新部署完Hexo之后，为了使博客能够提供更好的阅读体验，我尝试了Material Theme。当然，我先前使用的主题Yilia也是非常优秀的，其作者的博客为Litten。Material Theme提供了非常详细的设置指南，基本可以满足需要。我对前端一窍不通，通过两天的摸索，现在将遇到的问题以及设置文档不够详细的内容在此进行分享，希望帮助到想要使用该主题的你~

1 hexo g 报错

关于该问题的反馈可查看issues #686 hexo s 仍旧报错，这个应该是1.5.5和1.5.6版本中出现的问题。根据评论中@michaelcai给出方法可以解决该问题。方法如下：

进入theme/material/layout/_widget文件夹下，找到dnsprefetch.ejs文件，找到：

<% } else if(theme.comment.use.startsWith("disqus")) { %>
# 该段代码应该位于第21行

将该行代码修改为：

<% } else if(theme.comment.use && theme.comment.use.startsWith("disqus")) { %>

然后保存修改，执行hexo clean再执行hexo g应该就没问题了。

2 更改favicon、img下的相关图片

favicon是显示在网页标签左端的小图片。你可以将任意喜欢的图片放置于\themes\material\source\img文件夹下，建议设置一个简洁的名字，例如your.png；或者直接将名字设置为favicon.png以替代原本存在的文件，这样的好处是不需要到主题的配置文件_config.yml中修改favicon的指向了。

如果你选择的是第一种方式，假设你的favicon图片的名称为your.png，则打开主题的_config.yml文件，将favicon修改为：

favicon: "/img/your.png"

high_res_favicon以及apple_touch_icon同favicon的方法设置。

img下logo、avatar、和daily_pic所指代的图片如下图所示

其中，avatar还会显示在左边侧栏上方。对它们的修改方法同favicon，只需将相关图片放入\themes\material\source\img文件夹下，然后修改主题_config.yml文件中的相关项目，使其指向自己想要的图片即可。

3 为 url 下的 daily_pic 和 logo 设置跳转页面

material可以为首页的daily_pic和logo设置跳转链接，当你点击首页的daily_pic和logo时会跳转到你指定的网页。

在利用这个功能时，我首先写了两篇文章，并使用hexo g获得它们的html文件,然后将它们在\source\_posts下的.md文件删除（这样它们就不会显示在发布的文章之中）。将这两份html文件重命名成文章的题目（以便区分它们），然后在\public文件夹下新建一个文件夹(homestories)用于存放这两份html文件，然后将url下的代码指向这两份文件：

url:
    daily_pic: "/homestories/O-Alquimista.html"
    logo: "/homestories/the-little-prince(chapter-21).html"

这样的话，点击我博客首页的daily_pic和logo，你将会看到两篇在公布的文章中找不到的文章，它们也不能被博客的搜索引擎检索到。不过，由于这两份文件是直接添加到\public之中的，所以如果执行了hexo clean命令删除\public文件夹之后，再执行hexo g是无法生成它们的。所以需要将它们做好备份，当清除\public文件夹之后，就需要手动将它们添加回去。

当然，将logo的 url 指向你的 About 页面也是一个不错的主意。首先你得根据material的设置文档设置好了 About 页面（About 页面中需要展示的内容直接在生成 About页面的index.md中写）。然后将logo代码修改为：

url:
    logo: "/about/index.html"

4 设置版权声明 License

你可以直接在主题配置文件_config.yml对 license进行设置，该版权声明将会应用于你发布的全部文章中。你也可以在markdown文件的Front-matter中添加license字段，从而为该文章单独设置版权声明。

本博客所使用的版权声明样式来自于 hexo 折腾记|Tianma中关于版权声明的相关设置，你可以根据其中的相关内容配置你的版权声明。

5 为文章添加多个tags

在Front-matter中使用如下语法格式为文章添加多个 tags：

tags: 
- tag1
- tag2
- ······

6 按照设置指南创建“图库”页面会导致错误

我按照设置指南中的介绍创建“图库”页面并未成功，在查阅了众多相关文章之后亦未解决该问题。

7 使用不蒜子需要修改 busuanzi_pure_mini_js 的指向

由于不蒜子的域名更改了，所以在使用不蒜子提供的统计js时需要使用到新的地址：

busuanzi:
    enable: true
    all_site_uv: true
    post_pv: true
    busuanzi_pure_mini_js: "https://busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"

8 使用mathjax

请对mathjax作如下修改：

# MathJax 2.7.0-2.7.1
mathjax: https://cdn.bootcss.com/mathjax/2.7.0/MathJax.js

此外，你需要安装mathjax插件并进行一些配置，可参见我的另一篇博文末尾所提到的两篇文章。在需要使用mathjax的文章中添加mathjax标签

mathjax: true

9 为图片添加点击放大功能

请查看Hexo 折腾：利用 Fancybox 添加图片放大预览查看功能。

对于下载的fancybox库，只需要用到其\dist文件下的jquery.fancybox.min.css 和 jquery.fancybox.min.js。在/theme/material/source/js/文件夹下新建文件夹，命名为”fancybox”，然后将前面提到的两个文件复制到该文件夹下。之后便可按照上面网址所介绍的进行设置了。

10 设置文章中图片布局及其标题为居中

目前我并未找到一个好的解决方法（因为我对css一窍不通）。我当前的做法是不使用![描述](图片链接)格式来插入图片，而是使用html命令：

<div align="center">
<img src="图片的地址"  alt="图片说明" >
</div>
<center>图片的标题</center>

其中，<center>图片的标题</center>中的“图片的标题”是显示在图片正下方的。
